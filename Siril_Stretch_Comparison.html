What is "Stretching"?

An astronomical camera captures photons linearly: twice the photons = twice the pixel value. But most of the interesting nebula detail lives in the bottom 1–5% of that range, invisible on screen. "Stretching" is a nonlinear remapping that boosts those faint signals while compressing the bright stars so everything becomes visible. The four approaches below differ in which mathematical function they use, how they handle color, and how much manual control they offer.

BUILT-IN Siril Autostretch (GHT / MTF)

What it does

Siril's native autostretch uses a Midtone Transfer Function (MTF) — the same rational function used by PixInsight's STF (Screen Transfer Function). It computes image statistics (median, MAD), derives a black point and a midtone balance factor, then applies a single-pass curve to the whole image. Since Siril 1.2+ this is wrapped in the more general Generalised Hyperbolic Stretch (GHT) framework, but the default autostretch mode is still the classic MTF.

Core formula

MTF(x, m) = (m − 1) · x / ((2m − 1) · x − m) where m = midtone balance (0 < m < 1) x = pixel value normalized to [0, 1] after black-point subtraction

Pipeline

Compute median + MAD→ Derive shadow clip→ Solve for midtone m→ Apply MTF to all channels (linked)

Color handling

Channels are linked by default — one set of (shadows, midtones, highlights) applied equally to R, G, B. This preserves color balance from prior calibration (e.g., SPCC) but can't independently tune color response.

Character

Fast and predictable. Produces a "good first look" with neutral tonality. Limited highlight control — bright stars can clip. No HDR compression or iterative refinement. Purely automatic with no GUI sliders.

SCRIPT 1 AutoStretch Preview (Rich Stevenson — Deep Space Astro)

What it does

This script is essentially Siril's autostretch made interactive. It uses the exact same MTF math but wraps it in a PyQt6 GUI with live sliders for Target Median and Sigma. You can drag the sliders, see the result update in real time, then close the window — the image in Siril is untouched (preview only). It also adds a Link/Unlink Channels toggle for per-channel independent stretching.

Core formula

Identical to Siril built-in: MTF(x, m) = (m − 1) · x / ((2m − 1) · x − m) Difference: Target Median and Sigma are user-adjustable via sliders → sigma controls the black clip point: bp = median − σ · MAD → target_bg controls where the median lands after stretch

Pipeline

Compute per-channel median + MAD→ σ · MAD → shadow clip→ Inverse MTF → solve m for target→ Apply MTF (linked or per-channel)→ Live preview

Key differences from Siril built-in

  • Interactive — you dial in the exact black point and brightness before committing
  • Unlinked channels — each R/G/B channel can get its own shadow clip and midtone, useful for narrowband composites or data with a strong color cast
  • Preview only — non-destructive; doesn't modify the loaded image
  • Can display up to 3 images side-by-side (e.g., before/after comparison)

SCRIPT 2 Statistical Stretch (Seti Astro → ported by Cyril Richard)

What it does

A port of Seti Astro's Statistical Stretch (SAS Pro). Uses a different rational function that maps the image median directly to a user-chosen target value. Adds optional curves boost, HDR highlight compression, and a "Luminance Only" mode that stretches only the luminance channel while preserving original color ratios.

Core formula

SAS(x) = (median_rescaled − 1) · target · x / (median_rescaled · (target + x − 1) − target · x) where: x = (pixel − black_point) / (1 − black_point) median_rescaled = median of x target = desired output median (e.g. 0.25) Black point = median − σ · noise (robust lower-half sigma)

This is mathematically related to the MTF but parameterized differently: instead of solving for a midtone balance m, it directly maps the rescaled median to the target. The curve shape is similar but the black-point estimation uses a robust lower-half sigma (only pixels below the median contribute to the noise estimate), which is more resistant to nebula signal inflating the noise measurement.

Pipeline

Robust σ (lower-half)→ Black-point clip→ Median-targeted rational stretch→ Optional: Curves boost→ Optional: HDR compress

Extra features beyond the basic stretch

  • Curves Boost: A 6-point spline applied after the main stretch to selectively lift midtones while pinning shadows and highlights — gives more "pop" to nebula structure
  • HDR Highlight Compression: Hermite spline that smoothly rolls off everything above a knee point (default 0.75), taming blown-out star cores
  • Luminance-Only Mode: Extracts L (weighted sum via Rec.709/601/2020), stretches L independently, then re-applies the original color ratios. This prevents hue shifts during aggressive stretching. A blend slider lets you mix between the linked RGB result and the luminance-recombined result.
  • Linked vs Unlinked: Like the AutoStretch Preview, channels can be stretched independently

SCRIPT 3 VeraLux HyperMetric Stretch (Riccardo Paterniti)

What it does

A fundamentally different approach. Instead of a rational (MTF) function, VeraLux uses an Inverse Hyperbolic Sine (arcsinh) stretch — a logarithm-like curve that compresses extreme dynamic range more gracefully than MTF. The key innovation is decoupling luminance from chrominance: it stretches only the luminance channel, then reconstructs RGB by multiplying by the original color ratios (R/L, G/L, B/L). This is "True Color" preservation inspired by Dr. Roger N. Clark's methodology.

Core formula

IHS(x) = (arcsinh(D · x + b) − arcsinh(b)) / (arcsinh(D · 1 + b) − arcsinh(b)) where: D = stretch intensity (10^logD, typically 10¹ to 10⁵) b = protection parameter (controls shadow floor) x = luminance after anchor subtraction, in [0, 1] Color reconstruction: R_out = L_stretched · (R_input / L_input) G_out = L_stretched · (G_input / L_input) B_out = L_stretched · (B_input / L_input)

Pipeline

Normalize input→ Adaptive anchor (morphological)→ Extract L (sensor-weighted)→ Compute color ratios R/L, G/L, B/L→ arcsinh stretch on L→ Opt: Linear Expansion→ Shadow Convergence→ Rebuild RGB from ratios→ Color Grip hybrid blend

What makes it unique

  • arcsinh curve: Unlike MTF which has a fixed "S" shape, arcsinh is a smooth logarithmic-like function. At low D values it's nearly linear; at high D it behaves like log(x), compressing highlights far more gently than MTF. This makes it much harder to accidentally clip bright star cores.
  • Sensor Profiles: Instead of one-size-fits-all Rec.709 luminance weights, VeraLux includes real quantum-efficiency-derived weights for 25+ sensors (Sony IMX571, Canon EOS, Nikon, etc.). This means the luminance extraction matches your actual camera's spectral response.
  • Adaptive Anchor: Uses morphological analysis (not just median − σ·MAD) to find the true black point. More robust for images with gradients or very faint targets.
  • Color Grip + Shadow Convergence: A hybrid blending system. Color Grip mixes between the "true color" ratio-based output (100%) and a per-channel scalar stretch (0%). Shadow Convergence damps color noise in dark areas by pushing shadow color ratios toward neutral, weighted by a power function of luminance.
  • Two modes: "Scientific" gives full manual control over Log D, Linear Expansion, Color Grip, and Convergence. "Ready-to-Use" runs an automated pipeline with a single "Color Strategy" slider and zero-clipping smart scaling.
  • Smart Iterative Solver: The auto-calculator uses binary search (40 iterations) with a "Floating Sky Check" that simulates the post-stretch scaling to avoid black clipping.

When to Use Which?

Siril Built-in Autostretch

Use it for a fast first-look during stacking or pre-processing. Great for checking data quality without committing to a stretch. One click, no thinking required.

AutoStretch Preview

Use it when you want to tune the autostretch before applying it. The sigma slider lets you control exactly how much background gets clipped to black, and the target median controls the overall brightness. Unlink channels for narrowband composites with very different signal levels per channel.

Statistical Stretch

Use it when you want a more refined one-shot stretch with post-processing built in. The curves boost adds contrast without a separate step, and the HDR compression tames blown-out star cores. The Luminance-Only mode is valuable when you notice hue shifts in the standard linked stretch — it keeps nebula colors faithful by stretching only the brightness.

VeraLux HyperMetric

Use it for your final, publication-quality stretch — especially on targets with extreme dynamic range (M42 core, galaxy nuclei, bright+faint nebulae in one frame). The arcsinh curve naturally handles 4+ orders of magnitude without clipping. The sensor profiles matter most when doing broadband RGB with known astro cameras. The "Ready-to-Use" mode makes it almost as simple as the others; "Scientific" mode gives full control for experienced users.
